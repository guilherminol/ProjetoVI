Documento de Desenvolvimento: Sistema RAG de Suporte TécnicoProjeto: Automação de Suporte Nível 1 - Cancella Informática LTDAData: Maio de 2026 | Versão: 1.0Stack Principal: Python (FastAPI), LangGraph, PostgreSQL (pgvector), React.1. Objetivos TécnicosFinalidade: Reduzir em 20% a carga de tickets manuais de Nível 1.Abordagem: Retrieval-Augmented Generation (RAG) para evitar alucinações.Hospedagem: On-premise (Servidores locais da Cancella Informática).2. Requisitos do Sistema2.1. Requisitos Funcionais (RF)RF-01: Chatbot Semântico: Responder dúvidas técnicas com base em manuais via busca vetorial.RF-02: Gestão de Sessão: Manter contexto do histórico da conversa.RF-03: Download de Fontes: Link direto para o PDF original citado na resposta.RF-04: Feedback Binário: Botões de "Útil/Não Útil" para curadoria de dados.RF-05: Ingestão de Documentos: Interface/API para upload e processamento de PDFs legados.2.2. Requisitos Não Funcionais (RNF)RNF-01: Performance: Latência máxima de resposta de 5 segundos.RNF-02: Segurança: Logs de conversas identificados (auditoria técnica).RNF-03: Disponibilidade: Operação 24/7 para clientes B2B.RNF-04: Soberania de Dados: Execução local (Docker) sem dependência total de nuvem externa para armazenamento de documentos sensíveis.3. Stack Tecnológica e FerramentasBackend & IALinguagem: Python 3.11+Framework API: FastAPI (pela alta performance e tipagem).Orquestração de IA: LangGraph (para gerenciar fluxos de decisão e estados).Processamento de PDF: PyMuPDF ou Unstructured.Embeddings: OpenAI (Ada-002) ou modelos locais (HuggingFace) conforme latência.Banco de Dados (Camada de Dados)DB Principal: PostgreSQL 16+.Extensão Vetorial: pgvector (armazenamento e busca de documentos).FrontendFramework: React.js.Comunicação: Axios (Consumo de API REST).Estilização: Tailwind CSS (Responsividade para o Widget).Infraestrutura / DevOpsContainerização: Docker & Docker Compose.Servidor: Infraestrutura local da Cancella Informática (Windows/Linux Server).4. Arquitetura de Dados (Workflow RAG)Ingestão: PDF → Chunking (Quebra de texto) → Embedding (Vetorização) → PostgreSQL (pgvector).Recuperação: Pergunta do Usuário → Embedding da Pergunta → Busca por Similaridade de Cosseno no DB.Geração: Contexto Recuperado + Prompt de Sistema + Pergunta → LLM → Resposta Técnica.Orquestração: O LangGraph valida se a resposta está no contexto; se não estiver, direciona para "não encontrado" em vez de alucinar.5. Métricas de Sucesso (KPIs)Deflection Rate: % de problemas resolvidos sem intervenção humana.Response Time: Média de tempo entre o envio da dúvida e o início do streaming da resposta.Accuracy Rate: % de respostas validadas como corretas pelos técnicos sêniores.Nota de Risco: Documentos legados com baixa qualidade de OCR podem exigir uma etapa de pré-processamento de imagem/limpeza de texto antes da vetorização.
